FieldKit is an offline-capable form builder for NGOs and teams collecting data in the field — areas with unreliable or no internet.

Three components work together:

- **Serverside** — Web platform where you build forms, share them, and review responses in real-time
- **Local Server** — Run forms on a local Wi-Fi network so people can submit without internet
- **Mobile App** — Collect responses offline on your phone and sync when you're back online

The same form config works across all three — build once, use everywhere.

## Quick Start

### Deploy the platform

```bash
# Clone and start with Docker
git clone https://github.com/ultramenid/formit.git
cd formit
docker compose up -d
```

The web app runs at `http://localhost:3000`. Sign in with Google to start building forms.

### Install the local server

```bash
npm install -g @malichamdan/fieldkit-local-server
```

Run `fieldkit` to start. Import a form config through the web UI and start collecting responses offline.

### Set up the mobile app

Download the FieldKit mobile app, scan a form's QR code to import it, then collect responses anywhere — online or offline. Tap **Sync** when you're back on the network.

## Create a Form

### Adding fields

Open your dashboard and click **New Form**. Drag fields from the left panel onto the canvas:

- Text, email, number inputs
- Long text for paragraphs
- Dropdowns and multiple choice
- Date picker, file upload, star ratings

Click any field to edit its settings — label, placeholder, help text, and validation rules.

### Sharing

Once saved, open **Share** to:

- Copy a link for respondents
- Show a QR code to import into the mobile app
- Download the config JSON for the local server

### Responses

Open a form's **Responses** tab. New submissions appear in real-time — no need to refresh. You can:

- Filter by source (online, mobile, local server)
- Import data from an exported local server file
- Export all responses as CSV, JSON, or XLSX

## Local Server

### Installation

```bash
npm install -g @malichamdan/fieldkit-local-server
```

Verify:

```bash
fieldkit --version
```

### Import and serve

1. Run `fieldkit` — it starts the server and opens the admin web UI
2. In the admin panel, upload or paste your JSON config under **Import Config**
3. The form is live — any device on the same Wi-Fi can open and submit

<Callout type="info">
  If your firewall prompts for permission, allow it. Other devices need to reach your machine on the local network.
</Callout>

### Export data

Open the admin web UI, go to **Responses**, and click **Export** to download as CSV or JSON.

## Mobile App

### Import a form

Open the form on the web platform, click **Share**, and show its QR code. In the mobile app, tap the scan button and point at the code. The form downloads and appears in your list.

### Collect responses

Open a form, fill it out, and submit. Responses save to your phone immediately — even when you're completely offline.

### Sync when ready

<Steps>
  <Step num={1} title="Sync one form">
    Tap the sync icon on a form card to push its pending responses.
  </Step>
  <Step num={2} title="Sync All">
    Tap **Sync All** to push every pending form's responses at once.
  </Step>
</Steps>

Responses are uploaded with deduplication — syncing the same response twice won't create duplicates.

### Delete a form

Tap the trash icon on a form card to remove the local copy from your phone.

## Config Format

Every FieldKit form is defined by a single JSON config. It's the same format whether you're serving the form online, on a local network, or collecting responses on mobile.

### Schema

```json
{
  "formId": "uuid",
  "title": "Community Health Survey",
  "description": "Please fill out this form...",
  "version": 3,
  "exportedAt": "2026-05-14T09:00:00Z",
  "secret": "<mobile-sync-secret>",
  "_serverUrl": "https://fieldkit.example.com",
  "fields": [
    {
      "id": "f1",
      "type": "text",
      "label": "Full name",
      "placeholder": "Enter your name",
      "helpText": "",
      "required": true,
      "validation": { "minLength": 2, "maxLength": 100 }
    },
    {
      "id": "f2",
      "type": "select",
      "label": "District",
      "required": true,
      "options": ["North District", "South District", "East District"]
    }
  ],
  "settings": {
    "submitButtonText": "Submit",
    "confirmationMessage": "Thank you for your response.",
    "allowMultipleSubmissions": false
  }
}
```

### Field types

| Type | Settings |
|------|----------|
| `text` | Label, placeholder, help text, required, min/max length, regex pattern |
| `email` | Label, placeholder, help text, required |
| `number` | Label, placeholder, help text, required, min/max value |
| `textarea` | Label, placeholder, help text, required, min/max length |
| `select` | Label, options list, help text, required |
| `radio` | Label, options list, help text, required |
| `checkbox` | Label, options list, help text, required, max selections |
| `date` | Label, help text, required, min/max date |
| `file` | Label, help text, required, accepted types, max size |
| `rating` | Label, help text, required, max stars |

<Callout>
  The `secret` and `_serverUrl` fields are added at export time. They allow the mobile app to authenticate and know where to sync responses.
</Callout>

## API

### Mobile sync

#### Upload responses

`POST /api/mobile/responses`

Sends a batch of offline-collected responses to the server. Each response includes a `submissionId` for deduplication — sending the same batch twice is safe.

#### Upload files

`POST /api/mobile/upload`

Multipart file upload for photos and attachments collected in the mobile app. Validated for accepted file types.

#### Download config

`GET /api/forms/[id]/export`

Returns the full form config JSON with `secret` and `_serverUrl` embedded. The mobile app uses the secret to authenticate all sync requests.

### Deduplication

All response imports use a `(formId, submissionId)` compound unique constraint. If a response with the same submission ID already exists, it's skipped — the import returns `{ imported: number, duplicates: number }`.
